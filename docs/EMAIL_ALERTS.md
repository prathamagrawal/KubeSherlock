# Email Alerts for KubeSherlock

Email notifications are sent automatically by the watcher for HIGH and CRITICAL severity failures.

## Configuration

Add these variables to your `.env` file:

```bash
# Email alerts (optional)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USE_TLS=true
SMTP_USER=your-email@gmail.com
SMTP_PASSWORD=your-app-password
SMTP_FROM=kubesherlock@yourdomain.com
ALERT_EMAIL_TO=oncall@yourdomain.com,devops@yourdomain.com
```

### Gmail Setup

1. Enable 2FA on your Google account
2. Generate an [App Password](https://myaccount.google.com/apppasswords)
3. Use the app password in `SMTP_PASSWORD`

## Integration Example

```python
from agent.notifier import EmailConfig, EmailNotifier
from agent.severity import detect_severity
from agent.watcher import PodFailure
from agent.investigator import InvestigationResult

# Load config
email_config = EmailConfig.from_env()
notifier = EmailNotifier(email_config)

# After investigation completes
severity = detect_severity(result, failure)

# Send alert only for HIGH/CRITICAL
if severity in ["HIGH", "CRITICAL"]:
    notifier.send_alert(failure, result, severity)
```

## Severity Levels

| Level | Trigger | Example |
|---|---|---|
| **CRITICAL** | Resource exhaustion, 10+ restarts | OOMKilled, disk full |
| **HIGH** | Crash loops, config errors | CrashLoopBackOff, ImagePullBackOff |
| **MEDIUM** | Recoverable errors, 5+ restarts | Generic errors |
| **LOW** | Everything else | Unknown reasons |

Only HIGH and CRITICAL alerts trigger emails.

## Email Format

```
KubeSherlock Investigation Report

SEVERITY: HIGH
POD: production/api-server-xyz
REASON: CrashLoopBackOff
RESTART COUNT: 5
PROVIDER: anthropic
ITERATIONS: 3

ROOT CAUSE ANALYSIS:
Container is crashing due to missing environment variable DATABASE_URL

DIAGNOSTIC COMMANDS:
kubectl -n production describe pod api-server-xyz
kubectl -n production logs api-server-xyz --previous
kubectl -n production get events --field-selector involvedObject.name=api-server-xyz
kubectl -n production top pod api-server-xyz

---
Generated by KubeSherlock
```

## Retry Logic

- 3 attempts with exponential backoff (0s, 5s, 15s)
- Logs errors but never crashes the watcher
- Continues monitoring even if email fails
